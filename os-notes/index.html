    并发

    • 并发：指两个或多个事件在同一时间间隔内发生。这些事件宏观上是同时发生的，但微观上是交替发生的。

    • 虚拟是指把一个物理上的实体变为若干个逻辑上的对应物。物理实体是实际存在的，而逻辑上对应物是用户感受到的。
    • 虚拟技术

    • CPU上会运行两种程序，一种是操作系统内核程序（是整个系统的管理者），一种是应用程序

    问：操作系统分为用户态和内核态，当发生中断时，会由用户态转到内核态；riscv32有四个特权级：machine、supervision、user、virtual，操作系统用户态内核态与riscv特权级之间的关系及其软硬件交互的行为。

    答：操作系统的用户态和内核态对应着程序的两种执行模式，分别代表低权限（用户态）和高权限（内核态）的执行环境。RISC-V架构的特权级（Privilege Levels）与操作系统的用户态和内核态紧密相关，特别是在处理中断、异常或系统调用时。

    RISC-V 的四个特权级

    1. **Machine 模式 (M-mode)**：
      • RISC-V 最低层次的特权模式，通常由启动代码和硬件管理程序使用。它具有对所有硬件资源的完全访问权限。
      • 操作系统启动时，通常会在 M-mode 进行硬件初始化，之后切换到更低的特权级运行。
    2. **Supervisor 模式 (S-mode)**：
      • 操作系统内核运行在 S-mode 中，是典型的内核态（Kernel mode）。
      • S-mode 具有中等特权级，能够访问操作系统管理的硬件资源，但不会像 M-mode 那样直接操作底层硬件。
      • 操作系统内核负责处理来自用户态的系统调用、管理虚拟内存、处理中断等。
    3. **User 模式 (U-mode)**：
      • 用户态程序运行在 U-mode，权限最低，通常只能访问自己的虚拟地址空间，无法直接操作硬件资源。
      • 用户态的程序通过系统调用（如访问文件、网络等）请求内核执行特权操作，从而进入内核态（即 S-mode）。
    4. **Virtual 模式 (H-mode)**：
      • 这是 RISC-V 的虚拟化特权级，用于支持虚拟机的运行，但目前一般并不常见。

    用户态与内核态的切换

    • 中断与异常：当系统发生中断（如硬件中断）或异常（如非法操作）时，处理器会自动从 U-mode 切换到 S-mode（或 M-mode），进入操作系统内核，处理相应的事件。处理完毕后，系统再返回用户态继续执行。
    • 系统调用：当用户态程序需要访问特权资源时，会通过 ecall 指令进行系统调用，处理器会从 U-mode 切换到 S-mode，由操作系统内核执行相应的操作。

    硬件与软件的交互

    • 中断与异常处理：RISC-V 架构通过 CSR（Control and Status Registers）来管理特权级切换和中断处理。例如，mstatus 和 sstatus 寄存器控制当前的特权级和中断使能状态，mepc 和 sepc 寄存器存储发生异常时的返回地址，方便处理完异常后继续执行。
    • 页表管理与虚拟内存：操作系统内核通过 S-mode 管理虚拟内存（通过页表）。当用户态程序访问非法内存时，会触发页错误异常，内核捕捉并处理这个异常（例如终止程序或修正页表）。
    陷入指令

    陷入指令：应用程序主动地将CPU控制权还给操作系统内核。“系统调用”就是通过陷入指令完成的。

    外中断

    • 操作系统内核决定接下来让另一个应用程序上CPU运行
    • 时钟部件每隔一个时间片会给CPU发送一个时钟中断信号
      • 这是并发的前提条件

    中断机制的基本原理

    • 不同的中断信号，需要不同的中断处理程序来处理。当CPU检测到中断信号后，会根据中断信号的类型去查询“中断向量表”，以此来找到相应的中断处理程序在内存中存放的位置。

    总结

    • 在引入中断机制之后，才能让操作系统正常工作，才能实现并发。

    系统调用

    应用程序先将系统调用涉及到的参数放到寄存器中，然后执行一条内陷指令触发内中断，转入相应的中断处理程序–即系统调用的入口程序。

    • 应用程序—–（陷入指令）—–系统调用入口程序—–（根据寄存器中的参数判断用户需要那种系统调用服务）—–fork系统调用的处理程序

    • 传递系统调用参数-执行陷入指令（用户态）-执行相应的内请求核程序处理系统调用（核心态）-返回应用程序

    注意：

    1. 陷入指令是在用户态执行的。执行陷入指令之后立即引发一个内中断。使CPU进入核心态，陷入指令是非特权指令。
    2. 发出系统调用请求是在用户态，而对系统调用的相应处理在核心态下进行

    凡是与共享资源有关的操作，会直接影响到其他进程的操作，就一定需要操作系统介入，就需要通过系统调用来实现。

    • Ubuntu、CentOS这些Linux发行版，其开发团队的主要工作是实现非内核功能，而内核都是使用了Linux内核

      • 将操作系统的主要功能模块都作为系统内核，运行在核心态
      • 优点：高性能
      • 缺点：内核代码庞大，结构混乱，难以维护
    • 微内核
      • 只把最基本的功能保留在内核
      • 优点：内核功能少，结构清晰，方便维护
      • 缺点：需要频繁地在核心态和用户态之间切换，性能低

    4. 当管道读空时，读进程将阻塞，直到写进程往管道中写入数据，即可唤醒读进程
    5. 管道中的数据一旦被读出，就彻底消失。因此，当多个进程读同一个管道时，可能会错乱。对此通常有两个解决方案：
      1. 一个管道允许多个写进程，一个读进程
      2. 允许有多个写进程，多个读进程，但系统会让各个读进程轮流从管道中读数据

    虚拟内存的基本概念

    传统存储管理方式的特征、缺点

    • 一次性：作业必须一次性全部装入内存后才能开始运行

      这会导致两个问题

      • 作业很大时，不能全部装入内存，导致大作业无法运行
      • 当大量作用要求运行时，由于内存无法容纳所有作业，因此只有少量作业能运行，导致多道程序并发度下降

    • 驻留性：一旦作业被装入内存，就会一直驻留在内存中，直至作业运行结束。事实上，在一个时间段内，只需要访问作业的一小部分数据即可正常运行，这就导致了内存中会驻留大量的、暂时用不到的数据，浪费了宝贵的内存资源

    局部性原理

    虚拟内存的定义和特征

    虚拟内存的三个主要特征：

    • 多次性：无需在作业运行时一次性全部装入内存，而是允许被分成多次调入内存
    • 对换性：在作业运行时无需一直常驻在内存，而是允许在作业运行过程中，将作业换入、换出
    • 虚拟性：从逻辑上扩充了内存的容量，使用户看到的内存容量，远大于实际的容量

    如何实现虚拟内存技术

    • 虚拟内存的实现建立在离散分配的内存管理方式基础上
    • 传统和虚存的区别：
      • 在程序执行过程中，当访问的信息不在内存时，由操作系统负责将所需信息从外存调入内存，然后继续执行程序
      • 若内存空间不够，由操作系统负责将内存中暂时用不到的信息换出到外存

    从进程运行的安全性考虑，设备分配有两种方式：

    • 安全分配方式
      • 为一个进程分配一个设备后就将该进程阻塞，本次I/O完成后才将进程唤醒

      • 进程发出I/O请求后，系统为其分配I/O设备，进程可继续执行，之后还可以发出新的I/O请求。只有某个I/O请求得不到满足时才将进程阻塞

    • 计算机开机时需要进行一系列初始化的工作，这些初始化工作是通过执行初始化程序（自举程序）完成的

    • 初始化程序可以放在ROM中，ROM中的数据在出厂时就可写入了，并且以后不能再修改

    • 完整的自举程序放在磁盘的启动块上，启动块位于磁盘的固定位置

      ROM中只存放很小的“自举装入程序”，通过执行该程序就可以找到引导块，并将完整的“自举程序“读入内存，完成初始化

    固态硬盘（SSD）

    机械硬盘 vs 固态硬盘

    机械硬盘的原理：磁性物质

    SSD固态硬盘的原理：基于闪存flash技术

    固态硬盘的结构

    • 闪存翻译层：负责翻译逻辑块号，找到对应页（Page）
    • 存储介质：多个闪存芯片–每个芯片包含多个块–每个块包含多个页（Page）

    读写性能特性

    • 以页为单位读or写–相当于磁盘的“扇区”（注意，每个页只能写一次，若重复对同一页中的内容进行修改，需要先擦除整个块）

    • 以块为单位“擦除”，擦干净的块，其中每页都可以写一次，读无限次

      • 擦除行为

        擦除前，先将未改变的页迁移到新的块内，再把修改的页的内容写在新的块内，最后把原来那个块全部删掉。

        把逻辑块号进行重新映射，（X+0,X+n）映射到新的物理地址

        一个逻辑地址对应的物理地址是可能发生改变的，只不过闪存翻译层会将其映射到正确的物理地址

    • 支持随机访问，系统给定一个逻辑地址，闪存翻译层可通过电路迅速定位到对应的物理地址
